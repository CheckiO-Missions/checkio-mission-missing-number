Nie mam pojęcia, jak zacząć rozwiązywać tę misję.

w przypadku rozwiązania, które Panu proponuję, pomysł jest następujący. Ponieważ różnica między każdymi dwiema sąsiednimi liczbami w postępie arytmetycznym jest stała, musi Pan sprawdzić każde trzy elementy na posortowanej liście początkowej i znaleźć "lukę" - gdzie różnice między elementami w dwóch sąsiednich parach w triplecie nie są równe.

Ok, od czego powinienem zacząć?

Aby wykonać ten pomysł, Państwa lista powinna być posortowana. Proszę wykonać sortowanie w miejscu za pomocą metody listy sort(). 

items.sort()

Jak uzyskać co trzy sąsiadujące elementy listy?

Za pomocą funkcji for loop proszę iterować po elementach listy, pobierając ich indeksy z funkcji range(). 

for i in range(...):
    ...

Ponieważ będą Państwo pracować z trzema sąsiadującymi elementami wewnątrz pętli i aby uniknąć generowania indeksów, które są poza listą, należy iterować nie do samego końca listy, ale o dwa elementy mniej.

Jak to zrobić? Proszę znaleźć długość listy za pomocą wbudowanej funkcji len() i odjąć od niej 2. Wynik proszę wykorzystać jako argument funkcji range(). 

for i in range(len(items) - 2):
    ...

Jak uzyskać wartości trzech sąsiadujących elementów listy?

Aby rozwiązanie było bardziej czytelne, proszę utworzyć trzy zmienne first, second i third. Proszę użyć unpacking i slicing, aby przypisać wartości trzech sąsiadujących elementów z listy do zmiennych. 

f, s, t = items[i: i + 3]

Jak dokładnie przeprowadzić sprawdzanie różnic?

Cóż, to proste. Za pomocą operatorów if statement i not equal (!=) proszę porównać różnice między dwiema parami zmiennych. 

if t - s!= s - f:
    ...

Nasza lista jest posortowana rosnąco (pierwszy element z trzech jest najmniejszy, trzeci - największy), więc obie różnice są dodatnie.

Jak znaleźć brakujący element?

Jeśli poprzedni warunek to True ("luka" została znaleziona), należy odjąć różnicę s - f od t. 

t - (s - f)

Dlaczego więc to działa? Na przykład, mają Państwo posortowaną listę [1, 2, 4]. Jeśli odejmie Pan od 4 "normalny" krok dla tej progresji 2 - 1 == 1, otrzyma Pan poprawną odpowiedź 4 - 1 == 3.

Ale działa to również, jeśli "luka" znajduje się między drugim a pierwszym elementem! Mają Państwo listę [1, 3, 4]. Jeśli wykonają Państwo niezbędne odejmowanie - 4 - (3 - 1) == 4 - 2 == 2, ponownie otrzymają Państwo poprawną odpowiedź!

Jak więc powinno wyglądać moje ostateczne rozwiązanie?

Wystarczy zawinąć kod w wyrażenie funkcyjne, połączyć wszystkie powyższe instrukcje i zwrócić wynik ostatniego wyrażenia z odejmowaniem.

Oto całe rozwiązanie:

def missing_number(items: list[int]) -> int:

    items.sort()
    for i in range(len(items) - 2):
        f, s, t = items[i: i + 3]
        if t - s!= s - f:
            return t - (s - f)

W tym rozwiązaniu znajduje Pan krok i iteruje po poprawnej progresji. Jeśli element progresji nie znajduje się na początkowej liście - jest to odpowiedź. 

def missing_number(items: list[int]) -> int:

    first, *_, last = sorted(items)
    step = (last - first) // len(items)
    for i in range(first+step, last, step):
        if i not in items:
            return i

Suma pewnej liczby elementów progresji może być policzona przez formułę. Różnica między tą sumą a sumą elementów początkowej listy jest brakującym elementem. 

def missing_number(items: List[int]) -> int:   

    first, *_, last = sorted(items)
    total = (first + last) * (len(items) + 1) // 2

    return total - sum(items)
